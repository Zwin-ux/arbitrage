# Experience Simulation Checklist

Use this checklist when reviewing any Codex output. The question is not only “does it compile?” but also “what does the EXE feel like in the hands of a real user?”

## First-launch simulation

A first-time user should feel:
- guided
- safe
- intrigued
- not overwhelmed
- not sold to

They should understand within 30 seconds:
- this is Superior
- it is local-first
- paper mode is the default
- live is gated
- the machine has a clear next step

## Hangar simulation

The Hangar should feel like mission control.
A user should instantly see:
- machine readiness
- what has been completed
- what matters next
- what is locked
- how score / paper progress fits into the loop

If the Hangar feels like a generic dashboard, the change failed.

## Scanner simulation

The Scanner should feel like a machine with one visual center and one tactical job:
- show routes
- explain why they passed or failed
- stay truthful to available data
- never imply magic

If the scanner looks like a crypto heatmap toy, the change failed.

## Bot Bay / Loadout simulation

The user should feel like they are arming a machine.
They should understand:
- available slots
- bot family / strategy family
- unlock state
- what each slot is watching
- what is staged

If it feels like a settings form, the change failed.

## Score simulation

The Score screen should feel like:
- honest progress
- banked paper evidence
- unlock progression
- disciplined score attack

It should not feel like:
- fake brokerage PnL theater
- hype charts
- gamified nonsense

## Live Gate simulation

The Live Gate should feel:
- severe
- explicit
- trustworthy
- difficult to rush through

If it feels easy, vague, or promotional, the change failed.

## Replay Lab simulation

A future Replay Lab should feel like:
- film room
- mission replay
- “watch the machine think”
- “what would this bot have done?”

If it feels like just another table view, the change failed.

## Visual system simulation

Ask:
- Is there one focal point per screen?
- Is the visual hierarchy obvious?
- Does the geometry feel authored?
- Are colors semantic and restrained?
- Does it feel like Superior, not generic Qt?

## Final quality bar

If a real user would describe the app as:
- “serious”
- “clear”
- “distinct”
- “machine-like”
- “safe to learn in”

then the change is on the right track.
